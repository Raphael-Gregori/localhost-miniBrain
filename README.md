# localhost-miniBrain

A self-directed training plan to deploy and maintain a local AI assistant.

## What I'm building

A fully local AI assistant — no cloud, no external API, no data leaving the network.  
Users connect through a secure VPN tunnel and chat with AI models running on local hardware.

## Why

Most AI tools send your data to third-party servers.  
This project is about giving users a private, self-hosted alternative  
where they keep full control over their data.

## My starting point

Developer background (~7 years, C#/Unity, GDScript/Godot), daily Linux user.  
Learning Docker, networking, and VPN configuration from scratch alongside this deployment.

Planned stack: Ollama for model serving, Open WebUI as the chat interface,  
Caddy as a reverse proxy, and WireGuard for remote access — all new territory for me.
