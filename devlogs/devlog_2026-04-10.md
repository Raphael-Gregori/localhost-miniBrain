## What I did today

### systemd — start vs enable
- Discovered that `systemctl` doesn't work inside Distrobox (no systemd as PID 1)
- Practiced on the Bazzite host using the `bluetooth` service
- Understood the core distinction:
  - `start` = starts **right now**
  - `enable` = starts **at next boot**
  - The two are independent
- Observed that an `enabled` service can be restarted by other system dependencies (e.g. GNOME waking bluetooth back up)

### SSH — keys and permissions
- Understood the key pair concept: public key goes on the server, private key stays on your machine
- Checked `~/.ssh/` permissions:
  - `id_ed25519` → `600` (read/write for me only)
  - `id_ed25519.pub` → `644` (readable by everyone)
  - `.ssh/` directory corrected from `755` → `700`
- Understood why SSH silently refuses to work if permissions are too open

### Debian VM setup
- Verified AMD virtualization support (SVM): 16 compatible cores ✅
- Installed `virt-manager` + `libvirt` + `qemu-kvm` via `rpm-ostree` on Bazzite
- Enabled `libvirtd` service with `systemctl enable --now`
- Added user to `libvirt` group
- Created a Debian 12 "Bookworm" VM:
  - 2GB RAM, 2 CPUs, 20GB VirtIO disk
  - No desktop environment (server only)
  - SSH server installed during setup
- Successfully SSHed from Bazzite into the VM: `ssh llmadmin@192.168.122.23` ✅
